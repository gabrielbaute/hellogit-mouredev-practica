Guía de uso GIT

    Pasos generales

touch

Es un comando que podemos ejecutar desde la terminal bash y nos va a permitir crear un archivo (debemos indicar la extensión del archivo), su sintaxis sería:

touch filename

git checkout

Permite regresar al estado anterior de un archivo, la sintaxis sería:

git checkout filename

El comando checkout también puede regresar a commits anteriores, sin embargo al hacer eso (retornar a ramas previas), se perderán todas las ramas que se hicieron después de ese commit. Esto puede ser útil para revertir cambios y volver a un estado anterior en que un error o un conflicto en el código no estaba. Para ello debemos disponer del hash del commit:

git checkout #hash

Si se dispone de los hash de los commits perdidos, es posible volver a esa configuración del proyecto. Adicionalmente, se puede designar cualquiera de las ramas como el nuevo HEAD o cabecera del árbol de proyectos. Para ello deberemos situarnos en esa rama y tipear:

git checkout HEAD

git status

Permite consultar el estatus de todos los archivos en el directorio, la sintaxis sería:

git status

git log

Me muestra el historial de modificaciones registradas (commits).

git log

git reflog

Historial completo de interacciones que hayamos hecho en el git:

git reflog

git add

Este comando agrega un archivo para que se le realice un commit. Los archivos marcados como "u" significa que no están siendo rastreados, mientras que si se marcan con una "m" significa que el archivo (siendo rastreado) tiene una modificación, si se le agrega el "git add" ese archivo también pasará al commit. La sintaxis sería:

git add filename

Pueden escribirse en fila varios nombres de archivo:

git add filename1 filename2 ...

Incluso podría escribirse que se agregue todo el contenido del directorio, en cuyo caso sería:

git add .

git commit

Este comando permite realizar el commit y registrar las modificaciones hechas en el directorio, su sintaxis sería:

git commit -m "comentario"

git tag

Un tag es una etiqueta que podemos agregar al commit en donde estamos, y que se verá reflejada al consultar el log de git:

git tag nombre_del_tag

Por buena práctica, los tag's siempre deben escribirse todos en minúsculas y con guión bajo en lugar de espacios. Si sólo escrio "git tag" en la consola, me listará los tag's que he creado.

Alias

Con este comando podemos crear "rutas" o "comandos" personalizados, en base a los que usemos mucho, su sintaxis sería:

git config --global alias.name "conjunto de comandos"

Con el siguiente código, podemos crear un árbol simplificado de los commits realizados:

git config --global alias.tree "log --graph --decorate --all --oneline"

El resultado, al consultar el archivo .gitconfig, sería:

tree = log --graph --decorate --all --oneline

git ignore

Es un archivo que va a incluir la lista de ficheros y/o directorios que queremos que no sean incluidos en un commit. Debe estar ubicado en la raíz del proyecto. Sintaxis de creación:

touch .gitignore

Dentro del .gitignore se escriben los nombres y las ubicaciones de los ficheros. Tambbién puede escribirse de tal manera que, independientemente de la ubicación del fichero, el sistema lo ignore, empleando la expresión "**/":

**/filename

El .gitignore es incluido en el repositorio.

git diff

Nos muestra los cambios en el repositorio sin necesidad de hacer un commit.

git diff
